# Reusable Assets and Internal Libraries

As AI usage matures within a team, efficiency no longer comes from individual prompts but from **standardized, reusable assets**. These assets reduce repeated effort, enforce consistency, and ensure that best practices are applied uniformly across the organization.

Reusable assets transform AI from an ad hoc tool into a **systematized engineering capability**.

---

## Prompt Templates

Prompt templates provide predefined structures for common tasks. Instead of constructing prompts from scratch, developers reuse standardized formats that already encode best practices.

Benefits:

- Reduces prompt design time
- Ensures completeness (state, goal, constraints, output)
- Improves consistency across developers
- Lowers iteration count

**Example: Bug Fix Template**

```
State:
<code snippet>

Issue:
<describe the bug>

Constraints:
- minimal changes only
- do not modify API/schema
- preserve existing behavior

Output:
- fixed code
- root cause explanation
- edge cases to validate
```

**Example: Feature Development Template**

```
State:
<existing system context>

Goal:
<feature requirement>

Constraints:
- follow existing architecture
- handle edge cases
- ensure scalability

Output:
- step-by-step implementation
- validation scenarios
```

Templates act as **guardrails**, ensuring that critical information is always included.

## Code Snippets

Frequently used logic should be standardized as reusable code snippets rather than repeatedly generated by AI. This ensures correctness and consistency in critical areas.

Common candidates:

- Transaction handling patterns
- Error handling structures
- Validation utilities
- Logging formats

**Example: Transaction Pattern**

```javascript
await db.transaction(async (trx) => {
  await deductBalance(userId, amount, trx);
  await createBetRecord(betData, trx);
});
```

Benefits:

- Reduces risk in critical operations
- Avoids re-validating the same logic repeatedly
- Provides a reliable foundation for AI-generated extensions

## Query Patterns

Database queries are a frequent source of both bugs and performance issues. Standardizing query patterns ensures that common operations are both correct and optimized.

Patterns should cover:

- Aggregations (e.g., GGR calculation)
- Pagination
- Filtering with indexes
- Handling rollbacks and edge cases

**Example: GGR Pattern**

```sql
SELECT 
  SUM(CASE WHEN purpose = 'CasinoBet' THEN amount ELSE 0 END) -
  SUM(CASE WHEN purpose = 'CasinoBetRollback' THEN amount ELSE 0 END) AS totalBetAmount,
  SUM(CASE WHEN purpose = 'CasinoWin' THEN amount ELSE 0 END) AS totalWinAmount
FROM ledger
WHERE user_id = :userId
AND created_at BETWEEN :start AND :end;
```

Benefits:

- Prevents repeated logical mistakes
- Ensures performance alignment with indexing
- Reduces dependency on AI for critical queries

## Validation Strategies

Validation strategies define how generated or modified code is verified before being accepted. Instead of relying on ad hoc checks, teams standardize validation as part of the workflow.

Validation should include:

- Functional correctness
- Edge-case coverage
- Performance considerations
- Failure scenarios

**Example: Validation Prompt**

```
Given this implementation:

<code>

Provide:
- test cases (normal + edge cases)
- failure scenarios
- performance concerns
- validation checklist
```

**Example: Checklist Approach**

- Does it handle null or invalid input?
- Does it maintain transactional integrity?
- Does it scale with increased data volume?
- Does it follow existing patterns?

Benefits:

- Reduces risk of silent failures
- Standardizes quality checks
- Ensures consistency across teams

---

## Building an Internal Library Mindset

Reusable assets are most effective when treated as a **shared internal library**, not individual shortcuts.

Key practices:

- Store templates and patterns in a central repository
- Continuously refine based on real usage
- Version and update critical patterns (e.g., financial logic)
- Encourage team-wide adoption

Over time, this creates a system where:

- Common problems have predefined solutions
- AI usage becomes faster and more predictable
- Quality improves through standardization rather than individual effort

---

Reusable assets reduce randomness in AI-assisted development. By standardizing prompts, code patterns, queries, and validation strategies, teams move from **trial-and-error usage** to a **repeatable, reliable engineering process**.
